# users/ — персональные контексты пользователей

Если создать папку `users/<telegram_user_id>/` с собственным `CLAUDE.md`, этот
пользователь автоматически получает СВОЙ промт, свои `uploads/` и `samples/` —
его работа изолирована от остальных (удобно, когда ботом пользуются несколько
адвокатов с разными делами).

Пользователи без своей папки работают в корне рабочей директории с общим
`CLAUDE.md`. Содержимое `users/` в git не попадает (см. .gitignore).

В `samples/` кладите образцы документов адвоката — агенту можно велеть читать
их и писать в его стиле.
