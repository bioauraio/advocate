# Агент-адвокат — юридический ассистент в Telegram на Claude

Telegram-бот с AI-агентом «юрист РФ»: консультации по любым правовым вопросам,
работа с материалами дел, подготовка процессуальных документов (в т.ч. на
фирменном бланке адвоката .docx), приём файлов (.docx/.pdf) на анализ и отправка
готовых документов прямо в чат. Внутри — Claude (Anthropic) с файловыми
инструментами: агент сам читает базу знаний, тома дела и судебную статистику.

Сделан практикующими для практикующих: бот в таком виде реально сопровождал
уголовное дело (анализ 7 томов, ходатайства, статистика наказаний для позиции
защиты). Форк [claude-code-telegram](https://github.com/RichardAtCT/claude-code-telegram)
Ричарда Аткинсона (MIT) с доработками под юридическую практику.

## Что умеет

- **Консультации по праву РФ**: факты → нормы (статья/часть/пункт) → анализ → вывод.
- **Материалы дел**: кладёшь тома/документы в `cases/<номер>/` — агент навигирует
  по ним Grep'ом, не переполняя контекст.
- **Документы**: пишет ходатайства/жалобы/запросы, собирает .docx на бланке
  адвоката (логотип + реквизиты) и присылает файл в чат.
- **Приём файлов**: .docx/.pdf/.txt из чата сохраняются в `uploads/`, текст
  .docx извлекается автоматически.
- **Судебная статистика в комплекте**: `knowledge/sudstat_nakazaniya_po_statyam_2016-2024.csv`
  — по каждой статье/части УК РФ за 2016–2024: сколько осуждено, оправдано и какие
  наказания реально назначены (реальный/условный срок, штраф и т.д.). Агент
  отвечает на вопрос «что реально дают по статье X» цифрами.
- **Несколько пользователей**: whitelist по Telegram id; каждому можно дать
  изолированный контекст со своим промтом (`users/<id>/`).

## Быстрый старт

Нужны: Linux-сервер, Python 3.11+, Node 20+ (для .docx-бланков), аккаунт Claude
(подписка Pro/Max или API-ключ).

```bash
# 1. Код
git clone <этот-репозиторий> /opt/lawyer-agent
cd /opt/lawyer-agent

# 2. Зависимости бота (poetry кладёт venv в проект)
pipx install poetry
poetry config virtualenvs.in-project true
poetry install

# 3. Claude CLI + авторизация по подписке
# установка: https://claude.ai/code
claude auth login

# 4. Рабочая папка агента
mkdir -p /opt/lawyer-agent/workspace
cp -r agent-workspace/* /opt/lawyer-agent/workspace/

# 5. Конфиг
cp .env.example .env        # впиши токен бота и свой Telegram id
# путь в mcp_config.json поправь под своё расположение

# 6. (опционально) бланк адвоката
cd agent-workspace/tools && npm install docx
cp letterhead.example.json letterhead.json   # свои реквизиты
# положи логотип в workspace/assets/letterhead_logo.png

# 7. Запуск
poetry run claude-telegram-bot
# или systemd: см. deploy/lawyer-bot.service.example
```

Напиши боту в Telegram — неавторизованному пользователю он покажет его id,
добавь этот id в `ALLOWED_USERS`.

## Устройство

- `src/` — код бота (Python, python-telegram-bot + Claude SDK/CLI).
- `agent-workspace/` — шаблон рабочей папки агента:
  - `CLAUDE.md` — промт-персонаж юриста РФ (правь под себя);
  - `knowledge/` — общие справки, судебная статистика;
  - `cases/` — материалы дел (в git не попадают);
  - `users/` — персональные контексты пользователей (в git не попадают);
  - `tools/` — сборка .docx на бланке из markdown;
- `deploy/` — пример systemd-юнита;
- `docs/` — документация исходного проекта.

Агент работает в песочнице: файловые операции ограничены `APPROVED_DIRECTORY`,
инструменты — списком `CLAUDE_ALLOWED_TOOLS` (по умолчанию: чтение, запись
документов, отправка файлов в чат; без произвольного Bash).

## Данные и приватность

Материалы дел, загрузки, персональные контексты и реквизиты бланка исключены из
git (`.gitignore`) — репозиторий можно форкать и дорабатывать, не рискуя данными
доверителей. База сессий — локальный SQLite. Весь обмен с LLM идёт через API/CLI
Anthropic — учитывайте адвокатскую тайну и получайте согласие доверителя, когда
это требуется.

## Дисклеймер

Агент — инструмент подготовки и проверки гипотез, а не замена адвокату.
Финальную ответственность за документы и правовую позицию несёт адвокат.
Ответы модели могут содержать ошибки — проверяйте нормы по СПС в действующей
редакции.
